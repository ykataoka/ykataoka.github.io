Prius Challenge
Overview
Prius Challenge was the racing competition about maximizing mpg for Prius by data-driven strategy. This involves both data science(theory) and real driving skills(practice). I applied both two different machine learning approaches to identify the best strategy to achieve higher mpg. The analysis revealed, for example, where to use 'neutral'/'drive' or 'EVmode(on/off)'. As a result, our team got the award of 'Best MPG by Machine Learning'! Plus, we achieved 83mpg(w/o penalties), while Toyota dream team did 86mpg. Yeah, we were so close and I can say data-driven approach worked! This competition was held at Sonoma Raceway by Toyota Research Institute in Mar 3rd, 2017. Our team was composed of me(data scientist) and 4 other members of great drivers!
Problems
Q1. How do we identify best strategy to maximize mpg? Given the dataset, it is required to identify how use inputs (throttle, brake, gear and EV-mode)
Q2. How does data scientist deliver the strategy to drivers? This is not only data analytics but also teamwork. Delivering the result to driver is key part.
(Sonoma Raceway has tough uphill and downhill; 47[m] height. This makes this challenge hard.)
Solution
1. Comparison between high and low performance
Applying clustering technique to gps information, I have divided the race track into 150 pieces. Then, I visualized the average and variance of the 4 inputs(gear, brake, throttle, ev-mode) using high-performance dataset (top 30% mpg data) and low-performance dataset (Bottom 30% mpg data). This visualization helps driver to undersntand the strategy of how they use 4 inputs to get higher mpg.
2. Understanding remarkable point
Using 600 features (4inputs x 150 locations) and mpg data as the label, supervised learning is applied. Then, the feature importance is applied to identify the important inputs which highly affect mpg. In other words, they are the remakable points that drivers specially consider.

Mining Muscle Use Data for Fatigue Reduction in IndyCar
Overview
This research discusses how data analytics on muscle use in extreme racing conditions can be conducted to find actionable insights for the driver One of the important insights is how to minimize driver’s muscle fatigue during a race, because IndyCar has regulations forbidding the use of power steering. This research tackles two technological challenges: 1. data validation on noisy signal obtained from wearable device in extreme condition, 2. data cultivation to find actionable insights for the driver from heterogeneous racing data. First, we propose a data quality assessment technique, enabling the judgment of whether data is reliable or not. This qualitative analysis revealed that the data validation method works 99.5% accuracy to classify data as reliable or not. Second, we propose a data visualization tool based on unsupervised learning that enables the driver or mechanics to discover useful feedback. We identified and demonstrated several actionable insights, e.g., identifying potential relaxation points.
Term : 2016.03 - 2016.10
Software : python (sklearn, tensorflow, keras, pandas, numpy), html/js(bootstrap, Grafana, bokeh.js, Mapbox, node.js), InfluxDB

Solution
The idea of the data validation is that the actual sensor data is considered as unreliable data if the difference betweeen actual and predicted value are big. The prediction model uses car-telemetry data as features and predict the EMG value. This aims for prediction model to avoid the bad effect of noisy EMG values.
Data cultivation uses the unsupervised learning for driver or mechanics to understand tremendous data in more intuitive way.
In additino to data analytics, I also created data visualization WebUI for them. Data validation is shown in real-time dashboard and data cultivation is shown in post analytics dashboard.
Result
The data validation model has 99.48% accuracy to classify whether the EMG sensor data is reliable or unreliable. Yeah, it has good performance. (Please see the details on my paper.)
The data cultivation identified the locations where driver tends to have unstable muscle use and where driver can take rest more. This may be feedback for the driver to avoid muscle cramp.

Failsafe for Drone System : Nonlinear System Analytics and Contorl for Trirotor Drone system
Overview
What if a drone loses one control input by accident while flying? Since a ordinal drone system has 4 inputs, this research tackles the control problem of 3 inputs drone model, trirotor.
First, this research revealed the minimal number of inputs for hovering control from the perspective of nonlinear control system theory. The model analysis reveals that hovering control is impossible for any three-inputs drone system through the argument of Locally Asymptotically Stabilizability(LAS).
Second, this research proposed a nonlinear controller for an underactuated UAV having three inputs to realize “circle motion”, the motion that the attitude of UAV points to the center of circle while the center of mass moves on a circle orbit. The proposed method is redesigning the time-varying output zeroing controller discretely. This method allows output functions to be kept zero even though output functions are changed.
Problems
Q1. What is the limitation in controlling both position and attitude in 3D (SE(3)) ?
There are 6 states (12 states including velocity) that needs to be controlled for hovering. How many inputs are the minimum number of the control inputs for hovering control? Can this be revealed by mathematical proof?

Q2. What control can be done with only 3 inputs? Even with the 3 inputs, there must be meaningful motion that the nonlinear control can achieve such as circle motion mentioned above. How do we design the nonlinear controller to realize this motion with only 3 inputs.
Solution
1. Nonlinear System Analytics for input number revealation
Previously, M.Ishikawa and M.Sampei revealed a necessary condition for LAS that is “A system is not feedback stabilizable if the number of inputs is less than the dimension of the equilibria submanifold.” In order to apply this theorem to the generic model of drone system, I identified the dimension of the equlibria submanifold of it by using the general mathematical formulation of drone system.
2. Nonlinear Control Strategy for circle motion
The left figure shows the strategy for the circle motion control. First, the system focuses on controlling the (x,y,z) position control by Output Zeroing Control. Second, once it is completed, the remaining dynamics (Zero Dynamics) mainly causes instability to yaw angle. The idea to control yaw angle is changing the time-variant output function with maintaining the (x,y,z) position. In other words, this method enables to control Zero Dynamics.
Result
1. The following control limitation is proved.
Theorem : "Assume that UAV is composed of only a rigid body and propellers, then it is impossible to locally asymptotically stabilize a generic UAV model having less than three rotors inputs via continuous state feedback." The details can be found on my paper (IFAC'11) .
2. Circle Motion Control with only three inputs.
The result of the numerical simulation is shown on right movie. As you can see, first the (x,y,z) position control is realized. Second, changing the yaw-angle with maintaining the (x,y,z) position is realized. The details can be found on my paper (CDC'13) .

Smart Fleet Management System using IoT-driven Battery Prediction
Overview
I participated in Mercedes Benz Hackathon at Silicon Valley '16 as a data scientist. The challenge was to create the innovative fleet management system within 24 hours. I worked with people whom I met for the first time on site and successfully delivered the smart fleet management PoC. We decided to implement the future vision of fleet management based on the core battery prediction functionality. This prediction would be accurately conducted by machine leanring using IoT data and CAN data. Using this prediction, we proposed the new smart fleet system with 1. smart route planning, 2. real-time navigation feedback.
The result of the hackathon was 2nd prize among 120+ applicants. The juries judged the quality of both idea and implementation.
Term : 2016.11 (24 hours)
Software : python (sklearn, pandas, numpy), html/js(angular JS, Grafana, Mapbox), InfluxDB
Problems
Q. How to improve EV's battery prediction accuracy?
In general, stopping by charging stations is a loss in fleet management businesses. However, when planning the route, it looks like hard to predict battery consumption. (Based on my experience using Nissan Leaf for more than one year, the battery prediction functionality does not count the characteristics of road and can not accurately predict battery consumption.) I think this is because many factors affect EV's battery performance such as the weight of the cargo, weather, temperature, route (gradient of terrain, traffic), time and how you drive. What if all of these factors are available through IoT sensors? The prediction performance should be dramatically improved!
Q. If the system can predict EV's battery consumption more accurately, what does the future fleet management for delivering system look like? There are many use cases using battery prediction though, we decided to focus on two functions : 1. smart route planning, 2. real-time navigation feedback.
Solution
1. Smart Route Planning
Every car has different battery capacity and remaining battery. Each of drivers has different driving behavior. Depending on the combination of the driver and the car, the best routing path may differ. Some routes require the driver to drop by charging station. Some routes should be the most energy efficient route to avoid dropping by charging station. Some routes should be the fastest route due to Amazon Prime. Using battery prediction, the combination of car, driver and route can be optimally matched.
2. Real-time Navigation Feedback
The fleet manager should want to have the real-time monitoring tool to see each driver's performance. This tool visualizes the battery prediction and current value. While some driver follows original planning of the battery use, the other drivers may not be able to follow it. In this case, fleet manager can tell driver to change driver behavior or go to the charging station during lunch time.
Result
Our team, FleetWise, won the 2nd prize in this hackathon. It was very nice to meet new people, discuss the challenges in this industry and think about something new in the IoT and Automotice Era.
